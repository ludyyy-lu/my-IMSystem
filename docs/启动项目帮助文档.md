# 启动项目帮助文档

本文档用于快速启动 my-IMSystem 的开发环境，并完成前后端联调自检。

## 1. 运行模式说明

推荐使用以下模式：

- 基础设施（MySQL、Redis、etcd、Kafka）通过 Docker 启动。
- Go 微服务（auth、user、friend、chat、ws、api）在宿主机运行。
- 前端通过 Vite 开发服务器运行。

这样最方便调试代码和查看日志。

## 2. 前置条件

- Go >= 1.23
- Node.js >= 20
- Docker + Docker Compose v2

## 3. 端口与配置对齐（重要）

当前仓库约定使用 MySQL 端口 `3308`：

- [deploy/docker-compose.yaml](../deploy/docker-compose.yaml): `3308:3306`
- 各服务 MySQL DataSource: `localhost:3308`

## 4. 启动基础设施

在项目根目录执行：

```bash
docker compose -f deploy/docker-compose.yaml up -d mysql redis etcd zookeeper kafka
```

检查容器状态：

```bash
docker ps --filter name=im-
```

## 4.1 一键启动（推荐）

如果你不想每次手动开多个终端，可以直接使用项目脚本：

```bash
./scripts/dev-up.sh
```

该命令会自动：

- 启动基础设施容器（MySQL、Redis、etcd、Kafka 等）
- 启动 6 个 Go 服务（auth/user/friend/chat/ws/api）
- 启动前端 Vite 服务
- 记录 PID 和日志到 `.run/`

常用配套命令：

```bash
# 查看各服务状态
./scripts/dev-status.sh

# 停止所有本地服务（保留基础设施容器）
./scripts/dev-down.sh

# 停止所有服务 + 基础设施容器
./scripts/dev-down.sh --with-infra
```

日志目录：

- `.run/logs/auth-service.log`
- `.run/logs/user-service.log`
- `.run/logs/friend-service.log`
- `.run/logs/chat-service.log`
- `.run/logs/ws-gateway.log`
- `.run/logs/api-gateway.log`
- `.run/logs/frontend.log`

## 5. 创建 Kafka Topic

仅首次需要执行：

```bash
docker exec im-kafka kafka-topics --bootstrap-server localhost:9092 --create --topic im-chat-topic --partitions 3 --replication-factor 1
docker exec im-kafka kafka-topics --bootstrap-server localhost:9092 --create --topic im-friend-topic --partitions 3 --replication-factor 1
docker exec im-kafka kafka-topics --bootstrap-server localhost:9092 --list
```

如果提示 topic 已存在，可忽略。

## 6. 启动后端服务

按顺序在不同终端执行：

```bash
cd auth-service && go run auth.go
cd user-service && go run user.go
cd friend-service && go run friend.go
cd chat-service && go run message.go
cd ws-gateway && go run ws.go
cd api-gateway && go run api.go
```

建议顺序：先 auth，再 user/friend/chat，最后 ws/api。

## 7. 启动前端

在新终端执行：

```bash
cd frontend
npm install
npm run dev
```

默认访问地址：

- 前端: http://localhost:3000
- API 网关: http://localhost:8090
- WS 网关: ws://localhost:8888

## 8. 导入开发测试数据（可选但推荐）

如果你希望登录后立即看到好友、申请和聊天记录，可导入种子数据：

```bash
docker exec -i im-mysql mysql -uroot -proot123 im < deploy/mysql/init/dev_seed.sql
```

默认测试账号：

- alice / alice123
- bob / bob123

## 9. 联通性自检

### 9.1 端口监听检查

```bash
lsof -nP -iTCP -sTCP:LISTEN | grep -E ':(3000|8080|8081|8082|8083|8090|8888) '
```

### 9.2 接口快速检查

```bash
curl -s -X POST http://127.0.0.1:8090/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"alice123","device_id":"dev"}'
```

返回 `code: 200` 即表示 API 联通正常。

## 10. 常见问题

### 10.1 bad resolver state / auth.rpc not started

通常是 auth-service 未启动或未成功注册到 etcd。先确认：

- auth-service 启动日志包含 `Starting rpc server at 0.0.0.0:8080`
- etcd 容器正常运行

### 10.2 listen ... address already in use

端口被旧进程占用。先找进程再 kill：

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
kill -9 <PID>
```

### 10.3 前端提示 vite: command not found

先进入前端目录安装依赖：

```bash
cd frontend && npm install
```

### 10.4 登录失败或用户表异常

检查 MySQL 端口是否与配置一致（见第 3 节），然后确认数据库可连通：

```bash
mysql -h 127.0.0.1 -P 3308 -u goxygen -ppass im -e "SELECT 1"
```

### 10.5 前端页面空白、没有任何按钮

这通常是前端静态资源（`/assets/*.js`）与浏览器缓存不一致导致。

先强制刷新浏览器（macOS：`Cmd + Shift + R`），再检查是否有旧资源 404：

```bash
curl -I http://127.0.0.1:3000/index.html
curl -I http://127.0.0.1:3000/assets/
```

如果你是通过 Docker 运行前端，执行以下命令重建前端镜像并重启：

```bash
docker compose -f deploy/docker-compose.yaml build --no-cache frontend
docker compose -f deploy/docker-compose.yaml up -d frontend
```

说明：项目已将 nginx 配置为“`/assets` 缺失即 404、不回退到 `index.html`”，可避免脚本被错误返回 HTML 造成白屏。
