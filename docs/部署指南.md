# IM即时通讯系统 - 部署指南

## 一、环境准备

### 1.1 系统要求

**硬件要求：**
- CPU: 4 核心及以上
- 内存: 8GB 及以上
- 磁盘: 50GB 可用空间

**软件要求：**
- 操作系统: Linux (Ubuntu 20.04+) / MacOS / Windows 10+
- Go: 1.21 及以上版本
- Docker: 20.10 及以上版本
- Docker Compose: 2.0 及以上版本
- Git: 2.30 及以上版本

### 1.2 依赖服务版本

| 服务 | 版本 | 说明 |
|------|------|------|
| MySQL | 8.0+ | 数据持久化 |
| Redis | 7.0+ | 缓存与会话管理 |
| Kafka | 3.5+ | 消息队列 |
| Zookeeper | 3.8+ | Kafka 依赖 |
| Etcd | 3.5+ | 服务发现与配置管理 |

## 三、手动部署（生产环境）

### 3.1 安装 Go 环境

```bash
# 下载 Go 1.21+
wget https://go.dev/dl/go1.21.6.linux-amd64.tar.gz

# 解压到 /usr/local
sudo tar -C /usr/local -xzf go1.21.6.linux-amd64.tar.gz

# 配置环境变量
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPATH=$HOME/go' >> ~/.bashrc
source ~/.bashrc

# 验证安装
go version
# 输出：go version go1.21.6 linux/amd64
```

### 3.2 安装依赖服务

#### MySQL 安装

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server -y

# 启动 MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation

# 创建数据库和用户
mysql -u root -p
```

```sql
CREATE DATABASE im_auth CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE im_user CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE im_chat CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE im_friend CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'im_user'@'%' IDENTIFIED BY 'im_password_2026';
GRANT ALL PRIVILEGES ON im_auth.* TO 'im_user'@'%';
GRANT ALL PRIVILEGES ON im_user.* TO 'im_user'@'%';
GRANT ALL PRIVILEGES ON im_chat.* TO 'im_user'@'%';
GRANT ALL PRIVILEGES ON im_friend.* TO 'im_user'@'%';
FLUSH PRIVILEGES;
```

#### Redis 安装

```bash
# Ubuntu/Debian
sudo apt install redis-server -y

# 修改配置（生产环境建议设置密码）
sudo vim /etc/redis/redis.conf
# 找到 requirepass 行，设置密码
# requirepass your_strong_password

# 重启 Redis
sudo systemctl restart redis
sudo systemctl enable redis

# 验证
redis-cli ping
# 输出：PONG
```

#### Kafka 安装

```bash
# 下载 Kafka
wget https://downloads.apache.org/kafka/3.5.1/kafka_2.13-3.5.1.tgz
tar -xzf kafka_2.13-3.5.1.tgz
cd kafka_2.13-3.5.1

# 启动 Zookeeper
bin/zookeeper-server-start.sh config/zookeeper.properties &

# 启动 Kafka
bin/kafka-server-start.sh config/server.properties &

# 创建 Topic
bin/kafka-topics.sh --create --topic im-chat-topic --bootstrap-server localhost:9092 --partitions 16 --replication-factor 1
bin/kafka-topics.sh --create --topic im-friend-topic --bootstrap-server localhost:9092 --partitions 4 --replication-factor 1

# 验证 Topic
bin/kafka-topics.sh --list --bootstrap-server localhost:9092
```

```bash

wget https://github.com/etcd-io/etcd/releases/download/v3.5.9/etcd-v3.5.9-linux-amd64.tar.gz
```

### 3.4 配置文件

每个服务的配置文件在 `{service}/etc/config.yaml`，需要修改连接信息：

**auth-service/etc/config.yaml 示例：**
```yaml
Name: auth-service
Mode: prod

# RPC 配置
RpcServerConf:
  Port: 50051
  Timeout: 30000

# MySQL 配置

  Host: 127.0.0.1
  Port: 3306
  User: im_user
  Password: im_password_2026

  MaxIdleConns: 10
  MaxOpenConns: 100

  Host: 127.0.0.1:6379
  Password: your_strong_password
  DB: 0

  Secret: your_jwt_secret_key_min_32_chars_long
```

### 3.5 启动服务

**使用 systemd 管理服务（推荐）**

创建服务文件 `/etc/systemd/system/im-auth.service`：
```ini
[Unit]
Description=IM Auth Service
After=network.target mysql.service redis.service etcd.service

[Service]
Type=simple
User=im
WorkingDirectory=/home/im/my-IMSystem
ExecStart=/home/im/my-IMSystem/bin/auth-service -f /home/im/my-IMSystem/auth-service/etc/config.yaml
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

类似地创建其他服务文件，然后启动：
```bash
sudo systemctl daemon-reload
sudo systemctl start im-auth
sudo systemctl start im-user
sudo systemctl start im-chat
sudo systemctl start im-friend
sudo systemctl start im-ws-gateway

# 设置开机自启
sudo systemctl enable im-auth
sudo systemctl enable im-user
sudo systemctl enable im-chat
sudo systemctl enable im-friend
sudo systemctl enable im-ws-gateway

# 查看服务状态
sudo systemctl status im-auth
```

## 四、负载均衡与高可用

### 4.1 Nginx 反向代理

安装 Nginx：
```bash
sudo apt install nginx -y
```

配置文件 `/etc/nginx/sites-available/im-system`：
```nginx
# HTTP API 负载均衡
upstream ws_gateway {
    # 使用 IP Hash 保证 WebSocket 连接稳定性
    ip_hash;
    server 127.0.0.1:8080;
    server 127.0.0.1:8081;
    server 127.0.0.1:8082;
}

server {
    listen 80;
    server_name im.example.com;

    # HTTP API
    location /api/ {
        proxy_pass http://ws_gateway;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket
    location /ws {
        proxy_pass http://ws_gateway;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;  # 24 小时
    }

    # 健康检查
    location /health {
        proxy_pass http://ws_gateway;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/im-system /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4.2 MySQL 主从复制

**主库配置（/etc/mysql/mysql.conf.d/mysqld.cnf）：**
```ini
[mysqld]
server-id = 1
log-bin = mysql-bin
binlog-format = ROW
```

**从库配置：**
```ini
[mysqld]
server-id = 2
relay-log = relay-bin
read-only = 1
```

配置主从同步：
```sql
-- 主库创建复制用户
CREATE USER 'repl'@'%' IDENTIFIED BY 'repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';
FLUSH PRIVILEGES;

-- 查看主库状态
SHOW MASTER STATUS;
+------------------+----------+--------------+------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB |
+------------------+----------+--------------+------------------+
| mysql-bin.000001 |      154 |              |                  |
+------------------+----------+--------------+------------------+

-- 从库配置同步
CHANGE MASTER TO
    MASTER_HOST='主库IP',
    MASTER_USER='repl',
    MASTER_PASSWORD='repl_password',
    MASTER_LOG_FILE='mysql-bin.000001',
    MASTER_LOG_POS=154;

-- 启动从库同步
START SLAVE;

-- 检查状态
SHOW SLAVE STATUS\G
```

**应用层读写分离（GORM 配置）：**
```go
import "gorm.io/plugin/dbresolver"

db.Use(dbresolver.Register(dbresolver.Config{
    Sources:  []gorm.Dialector{mysql.Open("主库连接串")},
    Replicas: []gorm.Dialector{
        mysql.Open("从库1连接串"),
        mysql.Open("从库2连接串"),
    },
    Policy: dbresolver.RandomPolicy{},  // 随机选择从库
}))

// 读操作自动路由到从库
db.Model(&User{}).Where("id = ?", 123).First(&user)

// 写操作路由到主库
db.Create(&user)
```

```bash
# 查看 Consumer Lag
```

## 八、安全加固

### 8.1 HTTPS/WSS 配置

**Let's Encrypt 免费证书：**
```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d im.example.com
```

**Nginx HTTPS 配置：**
```nginx
server {
    listen 443 ssl http2;
    server_name im.example.com;

    ssl_certificate /etc/letsencrypt/live/im.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/im.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256';

    location /ws {
        proxy_pass http://ws_gateway;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 8.2 防火墙配置

```bash
# 只开放必要端口
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22      # SSH
sudo ufw allow 80      # HTTP
sudo ufw allow 443     # HTTPS
sudo ufw enable
```
